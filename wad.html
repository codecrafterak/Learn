HTML5 · Foundation

HTML Foundations & Document Structure

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. Every web page you see begins with an HTML document — a plain text file that the browser reads and renders into the visual page you interact with.

What is HTML? concept

HTML uses elements — defined by tags — to wrap content and give it meaning. Tags come in pairs: an opening tag <tagname> and a closing tag </tagname>. Some tags are self-closing (void elements) and don't need a closing tag, like <br>, <img>, and <input>.

💡 HTML5 Note

HTML5 is the current and living standard of HTML, introduced in 2014. It added semantic elements, multimedia support (audio/video), Canvas, SVG, drag-and-drop, local storage, and much more. Modern browsers fully support HTML5.

The Basic HTML5 Document Template

HTML
<!-- 1. DOCTYPE declaration — tells browser this is HTML5 -->
<!DOCTYPE html>

<!-- 2. Root element — wraps everything -->
<html lang="en">

  <!-- 3. Head: metadata, not shown on page -->
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Web Page</title>
    <link rel="stylesheet" href="styles.css">
  </head>

  <!-- 4. Body: visible page content -->
  <body>
    <h1>Hello, World!</h1>
    <p>This is a paragraph.</p>
  </body>

</html>

HTML Attributes attributes

Attributes provide additional information about HTML elements. They are always placed inside the opening tag and follow a name="value" pattern.

HTML — Attribute Examples
<!-- id: unique identifier for an element -->
<div id="main-content">...</div>

<!-- class: reusable identifier for CSS/JS -->
<p class="highlight large">Text</p>

<!-- style: inline CSS -->
<span style="color: red; font-weight: bold;">Red text</span>

<!-- title: tooltip text on hover -->
<abbr title="HyperText Markup Language">HTML</abbr>

<!-- data-*: custom data attributes -->
<div data-user-id="42" data-role="admin">...</div>

<!-- Boolean attributes (no value needed) -->
<input type="checkbox" checked disabled>

HTML5 · Navigation

Hyperlinks & Anchors

The anchor tag <a> is what makes the web a "web" — it connects pages together through hyperlinks. Links can point to other pages, file downloads, email addresses, phone numbers, or specific parts of the current page.

The Anchor Tag <a>

HTML — Links
<!-- Basic external link -->
<a href="https://www.example.com">Visit Example.com</a>

<!-- Opens in new tab -->
<a href="https://google.com" target="_blank" rel="noopener noreferrer">
  Google (New Tab)
</a>

<!-- Internal page link (relative URL) -->
<a href="about.html">About Page</a>

<!-- Anchor link (jump to section on same page) -->
<a href="#contact">Go to Contact Section</a>
<section id="contact">...</section>

<!-- Email link -->
<a href="mailto:hello@example.com?subject=Hi">Email Us</a>

<!-- Phone link (mobile-friendly) -->
<a href="tel:+1234567890">Call Us</a>

<!-- Download link -->
<a href="file.pdf" download="MyFile.pdf">Download PDF</a>

HTML5 · Tables

Tables

HTML tables display tabular data — data that has a natural row-and-column structure, like a spreadsheet. Tables should never be used for page layout (use CSS Flexbox or Grid instead). They are ideal for data comparison, schedules, pricing tables, and structured datasets.

HTML5 · Media

Images

The <img> tag embeds images into a page. It is a void (self-closing) element. HTML5 also introduced <picture> and <figure> for responsive and semantic image handling.

HTML — Images
<!-- Basic image: src and alt are required -->
<img src="photo.jpg" alt="A scenic mountain view" width="600" height="400">

<!-- figure + figcaption for semantic image -->
<figure>
  <img src="chart.png" alt="Sales chart for Q4 2024">
  <figcaption>Fig. 1 — Q4 2024 Sales Performance</figcaption>
</figure>

<!-- picture: responsive images for different screens -->
<picture>
  <!-- Use avif on supporting browsers -->
  <source srcset="image.avif" type="image/avif">
  <!-- Use webp on supporting browsers -->
  <source srcset="image.webp" type="image/webp">
  <!-- Mobile vs desktop variants -->
  <source media="(min-width:768px)" srcset="large.jpg">
  <!-- Fallback for older browsers -->
  <img src="image.jpg" alt="Description" loading="lazy">
</picture>

<!-- Image as a link -->
<a href="page.html">
  <img src="thumb.jpg" alt="Click to view page">
</a>

Key Image Attributes

HTML5 · Embedding

Iframes — Inline Frames

An <iframe> (inline frame) embeds another HTML document, website, video, or map inside your current page. The embedded content runs in a completely separate browsing context (its own document environment).

HTML — Iframes
<!-- Basic iframe -->
<iframe
  src="https://example.com"
  width="800"
  height="500"
  title="Embedded page"
>
  <!-- Fallback content for browsers that don't support iframes -->
  Your browser does not support iframes.
</iframe>

<!-- Google Maps embed -->
<iframe
  src="https://maps.google.com/maps?q=Paris&output=embed"
  width="600" height="400"
  loading="lazy"
  title="Map of Paris"
></iframe>

<!-- YouTube video embed -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560" height="315"
  allowfullscreen
  title="YouTube video player"
></iframe>

<!-- Sandboxed iframe (security restriction) -->
<iframe
  src="external-content.html"
  sandbox="allow-scripts allow-same-origin"
></iframe>

The sandbox Attribute — Security

ValueWhat it allows
sandbox (empty)Maximum restrictions — no scripts, no forms, no popups
allow-scriptsAllows JavaScript execution
allow-formsAllows form submission
allow-same-originAllows reading cookies and local storage if same origin
allow-popupsAllows opening popup windows
allow-top-navigationAllows navigation of parent frame
⚠️ Security Note

Always use sandbox when embedding third-party content and never embed content from untrusted sources without it. Also use title for accessibility.

HTML5 · API

Drag and Drop API

HTML5's native Drag and Drop API lets users click and drag elements to new positions or into drop zones. It uses the draggable attribute and a series of events to handle the interaction — no libraries needed.

HTML + JavaScript
<!-- HTML -->
<div id="item"
     draggable="true"
     ondragstart="dragStart(event)">
  Drag me!
</div>

<div id="dropzone"
     ondragover="dragOver(event)"
     ondrop="drop(event)">
  Drop here
</div>

<!-- JavaScript -->
function dragStart(e) {
  e.dataTransfer.setData('text', e.target.id);
}

function dragOver(e) {
  e.preventDefault(); // Allow drop
}

function drop(e) {
  e.preventDefault();
  const id = e.dataTransfer.getData('text');
  e.target.appendChild(
    document.getElementById(id)
  );
}

Drag Events

HTML5 · Graphics

Canvas API

The <canvas> element provides a bitmap drawing surface that you control entirely with JavaScript. Unlike SVG, Canvas is raster-based — you draw pixels. It's ideal for games, data visualizations, image manipulation, and animations.

HTML + JavaScript — Canvas Drawing
<canvas id="myCanvas"
  width="400" height="200">
  Canvas not supported.
</canvas>

<script>
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Rectangle
ctx.fillStyle = '#f0a500';
ctx.fillRect(10, 10, 100, 60);

// Stroke (outline only)
ctx.strokeStyle = '#78a9ff';
ctx.lineWidth = 2;
ctx.strokeRect(130, 10, 80, 60);

// Circle (arc)
ctx.beginPath();
ctx.arc(300, 40, 30, 0, Math.PI * 2);
ctx.fillStyle = '#6fdc8c';
ctx.fill();

// Line
ctx.beginPath();
ctx.moveTo(10, 100);
ctx.lineTo(390, 100);
ctx.strokeStyle = '#be95ff';
ctx.stroke();

// Text
ctx.font = 'bold 18px monospace';
ctx.fillStyle = '#e8eaf0';
ctx.fillText('Canvas!', 160, 150);

// Gradient
const grad = ctx.createLinearGradient(0,160,400,200);
grad.addColorStop(0, '#ff6b35');
grad.addColorStop(1, '#f0a500');
ctx.fillStyle = grad;
ctx.fillRect(0, 170, 400, 30);
</script>

HTML5 · Graphics

SVG — Scalable Vector Graphics

SVG is an XML-based vector image format. Unlike Canvas (pixels), SVG uses geometric shapes described in code — so they scale infinitely without losing quality. SVG is perfect for icons, logos, charts, illustrations, and UI decorations.

SVG — Shapes, Text, Path
<svg width="300" height="200"
     viewBox="0 0 300 200"
     xmlns="http://www.w3.org/2000/svg">

  <!-- Rectangle -->
  <rect x="10" y="10" width="80" height="50"
        fill="#f0a500" rx="8"/>

  <!-- Circle -->
  <circle cx="150" cy="35" r="25"
          fill="#78a9ff"/>

  <!-- Ellipse -->
  <ellipse cx="240" cy="35" rx="40" ry="20"
           fill="#6fdc8c"/>

  <!-- Line -->
  <line x1="10" y1="80" x2="290" y2="80"
        stroke="#be95ff" stroke-width="2"/>

  <!-- Polygon (star/shape) -->
  <polygon
    points="150,90 160,120 190,120
            165,138 175,168 150,150
            125,168 135,138 110,120
            140,120"
    fill="#ff6b35"/>

  <!-- Path (arbitrary shape) -->
  <path
    d="M10,180 Q150,130 290,180"
    stroke="#4ecdc4" fill="none"
    stroke-width="3"/>

  <!-- Text -->
  <text x="150" y="198"
        text-anchor="middle"
        fill="#e8eaf0" font-size="12">
    SVG Shapes
  </text>

</svg>

HTML5 · Media

Audio Controls

The <audio> element embeds sound content directly in HTML without any plugins. HTML5 supports MP3, OGG, and WAV formats. Multiple <source> tags let you provide fallbacks for different browser support levels.

HTML — Audio Element
<!-- Basic audio with controls -->
<audio controls>
  <source src="music.mp3" type="audio/mpeg">
  <source src="music.ogg" type="audio/ogg">
  Your browser does not support audio.
</audio>

<!-- Audio with all attributes -->
<audio
  src="podcast.mp3"
  controls              <!-- Show play/pause/volume UI -->
  autoplay             <!-- Start playing immediately (muted needed) -->
  loop                 <!-- Repeat forever -->
  muted                <!-- Start muted -->
  preload="auto"       <!-- none | metadata | auto -->
></audio>

<!-- JavaScript Audio Control -->
<script>
const audio = document.getElementById('myAudio');
audio.play();          // Play
audio.pause();         // Pause
audio.volume = 0.5;    // 0.0 to 1.0
audio.currentTime = 30; // Seek to 30s
audio.playbackRate = 1.5; // Speed
</script>

controlsBooleanShow browser's built-in audio player UI
autoplayBooleanAuto-play (requires muted in most browsers)
loopBooleanRepeat playback endlessly
mutedBooleanStart with audio muted
preloadnone/metadata/autoHow much to preload before play

Video Controls

The <video> element embeds video directly in HTML, replacing Flash entirely. It supports multiple formats, captions/subtitles via <track>, and full JavaScript control through the HTMLMediaElement API.

HTML — Video Element
<!-- Full-featured video element -->
<video
  width="720"
  height="405"
  poster="thumbnail.jpg"   <!-- Preview image -->
  controls                  <!-- Show player UI -->
  preload="metadata"
  playsinline               <!-- iOS inline play -->
>
  <!-- Multiple formats for compatibility -->
  <source src="video.webm" type="video/webm">
  <source src="video.mp4" type="video/mp4">
  <source src="video.ogv" type="video/ogg">

  <!-- Subtitles / captions -->
  <track
    kind="subtitles"
    src="subs-en.vtt"
    srclang="en"
    label="English"
    default
  >

  Fallback text if video unsupported.
</video>

<!-- Responsive video (CSS) -->
<style>
  .video-container {
    position: relative;

  }

</style>

HTML5 · Head

Meta Tags & Head Elements

The <head> contains metadata — information about the document that is not displayed to the user but is critical for browsers, search engines, and social media platforms.

HTML — Complete Head Section
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">

  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- SEO: description shown in search results -->
  <meta name="description" content="Learn HTML and CSS">

  <!-- Author, robots -->
  <meta name="author" content="Your Name">
  <meta name="robots" content="index, follow">

  <!-- Open Graph (Facebook, LinkedIn share) -->
  <meta property="og:title" content="Page Title">
  <meta property="og:image" content="thumbnail.jpg">

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">

  <!-- Favicon -->
  <link rel="icon" href="favicon.ico">
  <link rel="icon" href="icon.svg" type="image/svg+xml">

  <!-- External CSS -->
  <link rel="stylesheet" href="styles.css">

  <!-- Preload critical resource -->
  <link rel="preload" href="font.woff2" as="font" crossorigin>

  <!-- Structured data (JSON-LD) -->
  <script type="application/ld+json">

CSS3 · Stylesheets

Inline, Embedded & External Style Sheets

There are three ways to apply CSS to HTML. Each has a different priority in the cascade and different use cases. Understanding all three is foundational to controlling how styles are applied.

1. Inline Styles

Applied directly on an HTML element using the style attribute. Highest specificity. Hard to maintain.

HTML
<p style="
  color: red;
  font-size: 18px;
  font-weight: bold;
">
  Red bold text
</p>

<div style="
  background: #1a1a1a;
  padding: 20px;
  border-radius: 8px;
">
  ...
</div>
⚠️ Use sparingly

Hard to override and maintain. Use only for dynamic JS-injected styles.

2. Embedded (Internal)

Inside a <style> tag in the <head>. Good for single-page styles or email templates.

HTML
<head>
  <style>
    /* These rules apply
       only to this page */

    body {
      background: #f0f0f0;
      font-family: sans-serif;
    }

    h1 {
      color: #333;
    }

    .card {
      padding: 20px;
      border-radius: 8px;
    }
  </style>
</head>

3. External Stylesheet

Separate .css file linked with <link>. Best practice for all real projects — reusable across pages.

HTML + CSS
<!-- In HTML head -->
<link
  rel="stylesheet"
  href="styles.css"
>

/* styles.css — separate file */
body {
  margin: 0;
  font-family: Georgia, serif;
}

h1 { color: navy; }

/* Import another CSS file */
@import url('reset.css');
@import url('typography.css');
✓ Best Practice

External stylesheets are cached by browsers, reducing load time on repeat visits.

CSS3 · Cascade

Rule Cascading

The cascade is the algorithm CSS uses to determine which rule wins when multiple rules target the same element and property. It considers three factors in order: origin & importance, then specificity, then source order.

CSS — Cascade in Action
/* 1. ORIGIN ORDER (from least to most authoritative): */
/*    Browser defaults → User styles → Author styles → !important */

/* 2. SOURCE ORDER: Last rule wins when specificity is equal */
p { color: blue; }    /* loses — comes first */
p { color: red; }     /* wins — comes last */

/* 3. !important overrides everything (avoid overusing) */
p { color: blue !important; }  /* wins, even if less specific */

/* 4. Cascading with inheritance */
body { font-family: Georgia; }
/* All children inherit font-family unless overridden */
code { font-family: monospace; } /* Overrides inherited value */
Demo — Cascade Priority
① Browser default — black unstyled text (lowest priority)
② Author stylesheet rule — green text (medium priority)
③ More specific author rule — purple (wins over green)
④ !important — orange (overrides everything, avoid!)
CSS3 · Specificity

Specificity

Specificity is a score that determines which CSS rule wins when multiple rules target the same element. It's calculated as a 3-part number: ID — Class/Attribute/Pseudo-class — Type/Pseudo-element.

CSS — Specificity Calculation
/* Format: (IDs, Classes, Elements) → higher = wins */

*               /* (0,0,0) = 0   — universal */
p               /* (0,0,1) = 1   — element */
p.intro         /* (0,1,1) = 11  — element + class */
.nav .item      /* (0,2,0) = 20  — two classes */
#header         /* (1,0,0) = 100 — ID (very high!) */
#header .logo   /* (1,1,0) = 110 — ID + class */

/* Inline style: (1,0,0,0) = always wins except !important */
/* !important:   (1,0,0,0,0) = nuclear option */

/* Example conflict: which wins? */
.menu a         { color: blue; }  /* (0,1,1) = 11 */
nav ul li a     { color: red; }   /* (0,0,4) = 4  */
/* .menu a wins! Higher specificity. */
CSS3 · Inheritance

Inheritance

Inheritance is CSS's mechanism where certain property values set on parent elements are passed down to their children automatically. Not all properties inherit — mainly text-related ones do. You can explicitly control inheritance with keywords.

CSS — Inheritance
/* Inherited by default: */
/* color, font-family, font-size,
   font-weight, line-height,
   letter-spacing, text-align,
   list-style, cursor, visibility */

body {
  font-family: Georgia, serif;
  color: #333;
  /* All descendants inherit these */
}

/* NOT inherited by default: */
/* margin, padding, border, background,
   width, height, display, position */

/* Force inheritance with keywords: */
.child {
  /* inherit: take parent's value */
  color: inherit;

  /* initial: reset to CSS default */
  font-size: initial;

  /* unset: inherit if inherited prop,
            initial if not */
  border: unset;

  /* revert: browser default */
  display: revert;
}

CSS3 · Layout

The CSS Box Model

Every HTML element is a rectangular box composed of four layers: content, padding, border, and margin. Understanding the box model is essential for all CSS layout work.

CSS — Box Model
.box {
  /* Content dimensions */
  width: 200px;
  height: 100px;

  /* Padding: space inside border */
  padding: 20px;         /* all sides */
  padding: 10px 20px;    /* top/bot left/right */
  padding: 10px 20px 15px 25px; /* T R B L */

  /* Border */
  border: 2px solid #333;
  border-radius: 8px;

  /* Margin: space outside border */
  margin: 20px auto;  /* center horizontally */

  /* CRITICAL: box-sizing */
  /* content-box (default): padding and border
     ADD to the declared width */
  box-sizing: content-box;

  /* border-box (recommended!): padding and
     border are INCLUDED in width */
  box-sizing: border-box;
}

/* Apply border-box globally (best practice) */
*, *::before, *::after {
  box-sizing: border-box;
}

Border Images

CSS3's border-image property lets you use an image (or gradient!) as the border of any element. The image is divided into nine slices, mapped to the border regions. This enables decorative, custom borders impossible with plain colors.

CSS — Border Image & Regular Borders
/* ─── REGULAR BORDERS (CSS2) ─── */
.box {
  /* Shorthand: width style color */
  border: 2px solid #333;

  /* Individual sides */
  border-top: 4px dashed red;
  border-right: 2px dotted blue;
  border-bottom: 3px double green;
  border-left: 5px groove purple;

  /* Border styles: solid | dashed | dotted |
     double | groove | ridge | inset | outset | none */

  /* Rounded corners (CSS3) */
  border-radius: 8px;
  border-radius: 50%;           /* Circle */
  border-radius: 20px 5px;     /* TL/BR TR/BL */
  border-radius: 10px 20px 30px 40px; /* TL TR BR BL */
  /* Elliptical radius: horizontal/vertical */
  border-radius: 50px/25px;
}

/* ─── BORDER IMAGE (CSS3) ─── */
.border-image-box {

  border-image: url('border.png') 30 round;

CSS3 · Layout

Flexbox

Flexbox (Flexible Box Layout) is a one-dimensional layout model that excels at distributing space and aligning items in a row or column. It replaced float-based layouts and is ideal for navigation bars, card grids, centering, and component layouts.

CSS — Flexbox Properties
/* ─── CONTAINER properties ─── */
.flex-container {
  display: flex;          /* or inline-flex */

  /* Direction of main axis */
  flex-direction: row;          /* ← default */
  flex-direction: row-reverse;  /* → reverse */
  flex-direction: column;       /* ↓ column */
  flex-direction: column-reverse;

  /* Wrap: allow items to wrap to next line */
  flex-wrap: nowrap | wrap | wrap-reverse;

  /* Shorthand: direction wrap */
  flex-flow: row wrap;

  /* Align items on MAIN axis */
  justify-content: flex-start;    /* default */
  justify-content: flex-end;
  justify-content: center;
  justify-content: space-between;
  justify-content: space-around;
  justify-content: space-evenly;

  /* Align items on CROSS axis */
  align-items: stretch;     /* default */
  align-items: flex-start;
  align-items: flex-end;
  align-items: center;     /* vertical center! */
  align-items: baseline;

  /* Align wrapped rows */
  align-content: space-between;

  gap: 16px;                 /* gutters between items */
  gap: 16px 24px;            /* row-gap column-gap */
}

/* ─── ITEM properties ─── */
.flex-item {
  /* Grow: how much item grows if there's extra space */
  flex-grow: 1;     /* takes up remaining space */

  /* Shrink: how much item shrinks if there's no space */
  flex-shrink: 1;   /* default: can shrink */
  flex-shrink: 0;   /* don't shrink */

  /* Basis: initial main-axis size */
  flex-basis: auto;  /* or 200px, 30%, etc. */

  /* Shorthand: grow shrink basis */
  flex: 1;            /* = 1 1 0 */
  flex: none;         /* = 0 0 auto */
  flex: 0 0 200px;

  /* Self-alignment (overrides align-items) */
  align-self: center;

  /* Order (default 0, lower = earlier) */
  order: -1;  /* move item to start */
}

CSS3 · Custom Properties

CSS Custom Properties (Variables)

CSS Custom Properties (also called CSS Variables) let you store values in named variables that can be reused throughout a stylesheet. They are scoped, cascade like regular properties, and can be updated dynamically with JavaScript — enabling themes, responsive adjustments, and design systems.

CSS — Custom Properties
/* Declare on :root for global scope */
:root {
  /* Naming convention: --property-name */
  --color-primary: #f0a500;
  --color-bg: #0d0f14;
  --spacing-md: 16px;
  --border-radius: 8px;
  --font-heading: 'Playfair Display', serif;
  --shadow: 0 4px 16px rgba(0,0,0,0.3);
}

/* Use with var() */
.button {
  background: var(--color-primary);
  padding: var(--spacing-md);
  border-radius: var(--border-radius);
  box-shadow: var(--shadow);

  /* Fallback value */
  color: var(--color-text, #ffffff);
}

/* Override at a specific scope */
.dark-theme {
  --color-bg: #000;
  --color-text: #fff;
}

/* Dark mode via media query */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #121212;
    --color-text: #e0e0e0;
  }
}

/* JavaScript: read and modify CSS variables */

// Set: el.style.setProperty('--color-primary', '#ff0000')
CSS3 · Responsive

Media Queries

Media queries apply CSS rules conditionally based on the user's device characteristics — screen size, orientation, resolution, color scheme preference, and more. They are the foundation of responsive web design.

CSS — Media Queries
/* ─── SYNTAX ─── */
@media media-type and (feature: value) { ... }

/* ─── MEDIA TYPES ─── */
@media screen  { }  /* screens */
@media print   { }  /* printing */
@media all     { }  /* all devices (default) */

/* ─── BREAKPOINTS (mobile-first) ─── */
/* Default: mobile styles */
.container { padding: 16px; }

@media (min-width: 640px) {   /* sm: small tablets */
  .container { padding: 24px; }
}

@media (min-width: 768px) {   /* md: tablets */
  .grid { grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {  /* lg: desktop */
  .grid { grid-template-columns: repeat(3, 1fr); }
}

@media (min-width: 1280px) {  /* xl: large desktop */
  .container { max-width: 1200px; }
}

/* ─── ORIENTATION ─── */
@media (orientation: landscape) { }
@media (orientation: portrait) { }

/* ─── DARK MODE ─── */
@media (prefers-color-scheme: dark) { }

/* ─── REDUCE MOTION (accessibility) ─── */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.001ms !important; }
}

/* ─── HIGH RESOLUTION ─── */
@media (min-resolution: 2dppx) {  /* Retina */
  .logo { background-image: url('logo@2x.png'); }
}

/* ─── LOGICAL OPERATORS ─── */
@media (min-width: 768px) and (max-width: 1024px) { }
@media screen and (min-width: 768px) { }
@media (max-width: 480px), (orientation: portrait) { }
@media not print { }

/* ─── CONTAINER QUERIES (CSS4!) ─── */
.parent { container-type: inline-size; }
@container (min-width: 400px) {
  .card { flex-direction: row; }
}
